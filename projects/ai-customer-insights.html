Customer Feedback Insights Dashboard

A small, practical project that turns messy customer reviews/support notes into clear themes, trend charts, and a simple “what to fix next” table — using lightweight NLP + an Excel dashboard.

Type: Personal case study Focus: Themes + Trends + Actions Tools: Python · Excel · Visualization
View projects GitHub ↗

Problem

Feedback is valuable, but it’s usually scattered (reviews, tickets, notes) and inconsistent. The goal here was simple: convert text noise into something a team can actually act on.

What I built

  • A clean theme table (theme, count, examples, suggested action)
  • A trend view (which themes are rising/falling over time)
  • An Excel dashboard with filters for time, product area, and severity

How it works (lightweight + real)

  • Input: customer reviews / support notes in a CSV
  • Clean: remove duplicates, normalize text, handle missing fields
  • Theme extraction: keyword + simple clustering to group similar issues
  • Evidence: store 3–5 real examples per theme (so it’s explainable)
  • Output: dashboard-ready tables + a short summary block

Outputs

Final format: Theme → Trend → Evidence examples → Recommended action (priority).

Why it’s believable

This mirrors real analytics work: take raw messy inputs, make them consistent, produce an output table, and visualize it in a way stakeholders can use — without pretending it’s a huge “enterprise pipeline.”
